# Contribution

Processus de contribution recommande.

## Etapes

1. Creer une branche de travail descriptive.
2. Implementer la modification (code + tests + doc associee).
3. Valider localement backend/frontend.
4. Ouvrir une PR claire avec impact technique et plan de verification.

## Checklist PR

- [ ] Fonctionnalite testee localement.
- [ ] Pas de regression visible sur auth/capteurs/alertes.
- [ ] Documentation mise a jour.
- [ ] Captures ou logs si changement UX/API notable.
