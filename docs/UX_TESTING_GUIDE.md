# Guide de test UX

Guide pratique pour valider l'experience utilisateur de la plateforme web Aerium.

## 1. Objectifs

- Verifier la comprehension des parcours critiques.
- Mesurer la fluidite de navigation.
- Identifier les frictions sur auth, capteurs, alertes et rapports.

## 2. Parcours a tester

1. Inscription/connexion.
2. Creation d'un capteur.
3. Consultation dashboard et analytics.
4. Gestion d'une alerte (reconnaissance/resolution).
5. Export d'un rapport.
6. Creation d'une tache de maintenance.

## 3. Profils de test

- Profil A : utilisateur standard (metier).
- Profil B : administrateur.
- Profil C : nouvel utilisateur sans contexte technique.

## 4. Metriques a relever

- Temps d'execution par tache.
- Taux de reussite sans assistance.
- Nombre d'erreurs ou hesitations.
- Satisfaction percue (note 1 a 5).

## 5. Grille d'observation

Pour chaque tache :

- Objectif compris ?
- Etapes intuitives ?
- Terminologie claire ?
- Feedback systeme suffisant ?
- Erreurs recuperables ?

## 6. Seuils de validation recommandes

- >= 85 % de reussite autonome.
- <= 2 erreurs majeures par session complete.
- Score de satisfaction moyen >= 4/5.

## 7. Rapport de test

Inclure :

1. contexte (version, date, profil),
2. resultats quantitatifs,
3. verbatims importants,
4. priorisation des ameliorations (P1/P2/P3),
5. plan d'action court terme.

## 8. Bonnes pratiques

- Tester sur ecran desktop et mobile.
- Tester avec donnees vides et donnees riches.
- Rejouer les memes parcours apres corrections.
